FINAL DB — HARD LOCK POS TRUTH LAYER

Objective:
Permanently lock lv_line_item and lv_modifier as the ONLY POS truth sources.

These must NEVER be transformed, mutated, or reinterpreted.

---

PART 1 — CODE LOCK (BACKEND)

Files:
- server/services/loyverseImportV2.ts
- server/routes/analysisV3.ts
- server/services/* (any analysis layer)

Add global guard:

function assertPosTruthLayer(context: string) {
  throw new Error(`POS_TRUTH_LAYER_VIOLATION: ${context}`);
}

---

RULES:

1. lv_line_item:
- Must only come from Loyverse sync
- NEVER updated after insert
- NEVER aggregated into stored tables

2. lv_modifier:
- Must store ONE ROW per modifier selection per unit
- qty MUST ALWAYS = 1
- NEVER updated after insert
- NEVER grouped or deduplicated at storage level

---

PART 2 — DATABASE PROTECTION

Add DB constraints:

ALTER TABLE lv_modifier
ADD CONSTRAINT qty_must_be_one CHECK (qty = 1);

---

Prevent accidental updates:

CREATE OR REPLACE FUNCTION prevent_lv_update()
RETURNS trigger AS $$
BEGIN
  RAISE EXCEPTION 'POS_TRUTH_LAYER_VIOLATION: UPDATE not allowed on %', TG_TABLE_NAME;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER no_update_lv_line_item
BEFORE UPDATE ON lv_line_item
FOR EACH ROW EXECUTE FUNCTION prevent_lv_update();

CREATE TRIGGER no_update_lv_modifier
BEFORE UPDATE ON lv_modifier
FOR EACH ROW EXECUTE FUNCTION prevent_lv_update();

---

PART 3 — ANALYSIS IS READ-ONLY

All analysis routes:

- MUST be SELECT only
- MUST NOT write to:
  - lv_line_item
  - lv_modifier
