FINAL DB — FIX LOYVERSE MODIFIER SYNC (MULTI-QUANTITY)

Objective:
Ensure lv_modifier stores ONE row per modifier selection PER UNIT sold.

Problem:
For items with qty > 1 (e.g. Double Set x2), only 1 modifier row is saved instead of 2.

Result:
Modifier counts are underreported.

---

FILES TO EDIT:
- server/services/loyverseSync.ts  (or wherever receipt import happens)
- server/services/loyverseImport.ts (if separate)
- Any function that processes receipt line items + modifiers

---

IMPLEMENTATION:

Locate code that processes line items:

Example pattern (current — WRONG):

for (const item of receipt.line_items) {
  for (const modifier of item.modifiers) {
    insertModifier({
      receiptId,
      name: modifier.name,
      qty: 1
    });
  }
}

---

REPLACE WITH:

for (const item of receipt.line_items) {
  const qty = item.quantity || 1;

  for (let i = 0; i < qty; i++) {
    for (const modifier of item.modifiers || []) {
      insertModifier({
        receiptId: receipt.id,
        lineItemId: item.id,
        name: modifier.name,
        groupName: modifier.group_name || modifier.name,
        qty: 1,
        rawJson: modifier
      });
    }
  }
}

---

IMPORTANT RULES:

1. DO NOT multiply qty in SQL later
2. DO NOT store qty > 1 for modifiers
3. ALWAYS store 1 row per selection
4. This must mirror how a customer selects options
