PART A — TASKS → WORK REGISTER (Foundation)
A1) Upgrade DB schema (idempotent)
Modify/extend the existing tasks table (or create if it’s missing) to support a proper register.
Required fields:
id (uuid pk)
title (text)
description (text)  must support long multi-line
status (text) enum-like: draft | in_review | approved | in_progress | blocked | completed | archived
priority (text): low | medium | high | critical
area (text): operations | finance | purchasing | marketing | dev | compliance
assignee (text) (e.g. bob | codex | replit | cam | staff)
created_at timestamptz default now
updated_at timestamptz default now
due_at timestamptz nullable
deleted_at timestamptz nullable (soft delete)
Indexes:
(status, updated_at)
(assignee, status)
(area, status)
(deleted_at)
If the current description is a short varchar, upgrade to TEXT.
A2) Task Activity Log (audit trail)
Create table task_activity (idempotent):
id uuid pk
task_id uuid fk
created_at timestamptz default now
actor text (bob/cam/replit/codex/system)
type text (created|updated|comment|status_change|assigned|archived|restored)
message text
payload jsonb default {}
Every create/update/status change should write an activity entry.
A3) Comments (refinement)
Create table task_comments (idempotent):
id uuid pk
task_id uuid fk
created_at timestamptz default now
author text
body text
A4) API endpoints
Add/upgrade endpoints (use existing routing conventions under /api/...):
GET /api/tasks
filters: status, priority, area, assignee, q(search), includeArchived
default exclude deleted_at IS NOT NULL
POST /api/tasks
creates task + activity log
GET /api/tasks/:id
returns full task + activity + comments
PUT /api/tasks/:id
update fields + activity entries
POST /api/tasks/:id/comment
add comment + activity entry
POST /api/tasks/:id/archive
set deleted_at=now, status=archived, activity entry
POST /api/tasks/:id/restore
set deleted_at=null, restore prior status or draft, activity entry
No hard delete.
